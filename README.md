# 视频排贴命名工具 v1.1

专为 Facebook 资源视频排贴命名设计的桌面工具。

## 功能特点

- 按日期 + 帖次 + 时间自动批量命名视频文件
- 支持自定义每日发帖数（1 ~ 8 贴）及随机时间波动范围
- 内置日历选择器，精准控制起始发布日期
- 支持按文件创建时间排序 / 随机打乱两种模式
- 两步重命名机制，彻底解决文件名碰撞问题
- 亮色 / 暗色双主题，胶囊滑块一键切换
- 新增命名模式：模式 A 完整重命名 / 模式 B 保留原文件名

## 下载使用

前往 [Releases](../../releases) 页面下载对应平台的版本：

| 平台 | 下载文件 | 使用方式 |
|------|---------|---------|
| Windows | `视频排贴命名工具-windows.exe` | 直接双击运行 |
| macOS | `视频排贴命名工具-macos.zip` | 解压后双击 `.app` 运行 |
| Linux | `视频排贴命名工具-linux.zip` | 解压后运行 `./视频排贴命名工具-linux` |

## 如何发布新版本

本项目使用 GitHub Actions 自动构建和发布，每次发布只需创建一个版本 Tag 并推送即可。

### 发布步骤

#### 1. 确保代码已提交并推送

```bash
# 查看当前状态
git status

# 添加所有改动
git add .

# 提交改动
git commit -m "你的改动说明"

# 推送到 GitHub
git push origin main
```

#### 2. 创建版本 Tag

版本号格式为 `v主版本.次版本.修订版本`，例如 `v1.0.0`、`v1.1.0`。

```bash
# 创建新版本 tag（将 v1.0.1 替换为你想要的版本号）
git tag -a v1.0.1 -m "Release version 1.0.1"
```

#### 3. 推送 Tag 触发自动构建

```bash
git push origin v1.0.1
```

推送后，GitHub Actions 会自动在 Windows、macOS、Linux 三个平台上构建，并生成带安全签名的 Release。

#### 4. 查看构建结果

- 构建进度：项目 **Actions** 页面
- 发布结果：项目 **Releases** 页面

### 版本号说明

| 版本号格式 | 什么时候用 | 示例 |
|-----------|-----------|------|
| `vX.0.0` | 重大更新、不兼容改动 | `v2.0.0` |
| `vX.Y.0` | 新增功能 | `v1.1.0` |
| `vX.Y.Z` | 修复 bug | `v1.0.1` |

### 如果构建失败怎么办

1. 访问项目 **Actions** 页面查看错误日志
2. 修复代码问题并推送
3. 删除失败的 tag 并重新创建：

```bash
# 删除本地 tag
git tag -d v1.0.1

# 删除远程 tag
git push origin :refs/tags/v1.0.1

# 重新创建并推送
git tag -a v1.0.1 -m "Release version 1.0.1"
git push origin v1.0.1
```
